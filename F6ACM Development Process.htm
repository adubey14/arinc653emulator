F6ACM Setup

1. Install ACM modeling tool-suite on the VM/ Wine

 
- Obtain the tool chain from https://f6.isis.vanderbilt.edu/redmine/dmsf/f6distro?folder_id=2
The latest version is dated March 22, 2012.
- Login to the VM as f6demo.

 

a) Initial Steps
-         Copy the GME and F6ACM installer to the VM. 
-         Uninstall any previous versions of GME/ F6ACM : Refer Un-Install Section below
-         Install the new versions of GME and F6ACM : Refer Install Section below.
 
b) Install
1.   Launch Wine Explorer
-         Open a Terminal
-         Type the command 
 wine explorer
 
2.   Install GME.
-         From Wine Explorer, double-click and launch the GME installer and run through the set up process.
-         Close Wine Explorer (This is required so that the installation is updated and visible in Wine).
 
3.   Install F6ACM
-         Unzip the F6ACM zip file to get the installer (.exe).
-         Launch the wine explorer as mentioned above.
-         From Wine explorer, double-click and launch the F6ACM installer.
-         Close Wine Explorer. (This is required so that the installation is updated and visible in Wine).
 
4.   Completing F6ACM installation.
ACM paradigm and F6ACM interpreter do not get automatically registered on Wine platform.
     Launch GME from the VM Menu (Applications->Wine->Programs->GME->GME).
 
     To Register the Paradigm, 
-      Register ACM Paradigm in GME (Tools + Register Paradigm + Add From File)
-      Browse to Paradigm folder in F6ACM Installation ($ACM_ROOT/Paradigm) (Default: C:\Program Files\ISIS\F6ACM\Paradigm). The C drive will be visible through My computer.
-      Load the file ACM.xmp to register the ACM Paradigm.
-      Right click on the dialog box to close it. The 'x' button might not be visible on wine.
 
 
     To Register the Interpreter, 
-       Click on Tools + Register Paradigm 
-       Select ACM Paradigm from the List.
-       Click on "Component" button.
-       If F6ACM is in the list of installed components, hit Close or right click on the dialog box to close it.
-       If F6ACM is not seen in the list of installed components, click on "Install New" button.
-       Browse to bin folder in F6ACM Installation ($ACM_ROOT/bin) (Default: C:\Program Files\ISIS\F6ACM\bin).
-       Load the file F6ACM.dll to register the F6ACM interpreter.
 
c) Un-Install
1.      Prior to uninstalling GME or F6ACM, export any existing models from mga files to xme files.
-         Open mga model file in GME.
-         Export to an xme file ( Click on File +Export)
2.      Launch Wine Uninstallers
-         Open a Terminal
-         Type the command 
wine uninstaller
3.      From the list of applications, choose the application that you wish to uninstall and run through the steps.

3. Setting up the VMs

 

LDEV and LEMU

The development VMs already contain the binaries for LDEV and Lemuapp.

Start network automatically using ncf script

To ease the setup of the network configuration a Python script called ncf was developped. Based on a configuration file or input arguments it manipulates system config files containing the name of the machine (/etc/hostname), the description of the network interfaces (/etc/network/interfaces), the IP address - hostname resolution (/etc/hosts), the NFS shares of an NFS server (/etc/exports) and the NFS mounts of an NFS client (/etc/fstab). In addition to the config file manipulation, the script changes the state of the machine in order to change its name (hostname), to restart the network based on the new configuration, to (re)start the mesh network (if requested), to (re)start the NFS service (if requested).

The test scenario which is intended to be built up using the script with appropriate settings contains four machines in the following roles:

  • VM named f6-vm0 is intended to be the server in the emulated radio mesh network. To this VM will connect the other VMs (mesh network members) when they set up their mesh network. NFS will not be supported on this machine. This VM has to be started first if mesh network is needed otherwise there is no need to have a VM called f6-vm0.
  • VM named f6-vm1 is intended to be a member of the mesh network and the NFS server that shares its /home and /opt folders to the NFS clients. This VM has to be started before the NFS clients. This VM can run without mesh network and/or NFS service, too.
  • VM named f6-vm2 is intended to be a member of the mesh network and the NFS client that mounts the shared folders described above. This VM can run without mesh network and/or NFS service, too.
  • VM named f6-vm3 is intended to be a member of the mesh network and the NFS client that mounts the shared folders described above. This VM can run without mesh network and/or NFS service, too.

The script has to be executed with root privileges otherwise you receive the following message:

 
Note these paths on your vm will be different and will depend upon the location of F6ACM inside F6demo home.
 
f6demo@f6development:~/$ ./ncf
Script has to be executed with root privileges, execute "sudo -s" to become root

The script can be used to achieve the following four goals:

  • Get information about usage or version of the script
  • Set up the network based on input arguments
  • Restore the system status to the last stored
  • Revert the system to the original state

These four modes will be described in detail below.

Get information about usage or version of the script

Help mode can be reached by executing ncf -h or ncf --help. It targets to give the user help by showing all possible valid inputs with details to understand the usage of the script.

 
root@f6development:~/F6ACM# ./ncf -h
ncf - netconfig script 2012-03-22
 
usage: ./ncf
 ./ncf -r
 ./ncf -<n> -[mn]
 
 -r (optional) revert all setting to original (no other options accepted if set)
 -<n> (mandatory) choose one of 0/1/2/3: name of the local machine (e.g., 0 for f6-vm0)
 -m (optional) if set: emulated radio mesh network will be set up (default is no mesh)
 -n (optional) if set: NFS share between VMs will be set up (default is no NFS),
 vm1 acts as server, vm2/3 act as clients, vm0 will NOT use NFS
 
examples:
./ncf -> set up VM similar to last successful execution
./ncf -r -> revert all settings
./ncf -0 -m -> f6-vm0, mesh used, no NFS
./ncf -1 -mn -> f6-vm1, mesh used, NFS server
./ncf -2 -n -> f6-vm2, no mesh used, NFS client
./ncf -3 -> f6-vm3, no mesh, no NFS
 
root@f6development:~/F6ACM#

The version of the script shows up in the first line of the help mode, there is an additional way to get just the version of the script by executing ncf -v or ncf --version.

 
root@f6development:~/F6ACM# ./ncf -v
ncf - netconfig script 2012-03-22
root@f6development:~/F6ACM#

Set up the network based on input arguments

In case we want to set up the test network based on input arguments, the script ncf expects one mandatory argument and two optional arguments. The mandatory argument is the name of the virtual machine on which the script is executed. The optional arguments refer to the mesh network and the use of NFS, for both it holds that default setting is not to use them if they are not passed to the script. From the name given the script deduces which roles the VM will play for the mesh and NFS services if any of these is set. If the arguments for mesh and/or NFS are not passed the script will not set it/them up.

The script changes the hostname of the VM, the configuration of the network interface eth0 and the /etc/hosts file that serves to provide a name to IP address resolution for the applications.
To set up the mesh network the following steps are executed depending from the VM. On f6-vm0 the link emulation server is started from /home/f6demo/LemuServer/LemuApp. On any other machine, first, the script tries to insert the kernel module ldev into the kernel. This kernel module is configured first, it is passed the IP address of the link emulator server (f6-vm0). Next, the virtual interfaces wlan0 and mesh0 are started and configured. If these steps succeed the VM has a working emulated radio connection to the other mesh network members.
To set up the NFS service the appropriate config file has to be manipulated and the service will be started. On the server machine, the file /etc/exports is changed to contain the folders to be shared together with the options and NFS server daemon is launched. On the client machines, the file /etc/fstab is changed to contain the folders to be mounted and the mount operation is performed.

NOTE: The script used with input arguments sets up a test network and connectivity is provided only between the guest machines and to the host machine. The connection beyond the host machine will be restored by using the script to revert the system (ncf -r).

Restore the system state to the last stored

This is used to run the last known good configuration. This configuration is stored /var/tmp

to use this run

 
$ ./ncf

Revert the system to the original state

 
$./ncf- r

Start LDEV and LEMU Manually.

We start now the lemu server to allow connecting the ldev radios. All traffic received by lemu server from a single radio will be automatically passed to all of the other attached radios.

 
./LemuApp

LemuApp will automatically attach to the address 0.0.0.0 (port 1250) when started.

You have to know the IP address of the lemu server (LemuApp) to be able to start LDEV. The mac address of the node can be randomly generated (if you omit the line) or specified using the mac0 option below. The following step has to be executed on each (virtual) machine that will take part as a member in the radio mesh network.

 
modprobe mac80211
insmod ldev.ko
cat <<EOF > /proc/ldev
ipaddress = xx.xx.xx.xx
mac0 = yy:yy:yy:yy:yy:yy
radio = add
EOF

Setting up the mesh network

The following part will describe how to set up a single mesh network. You can set up multiple mesh networks (with more or less members) then simply use another parameters with the same commands of the following steps.

 
sudo ifconfig wlan0 up
sudo iwconfig wlan0 channel 1
sudo ifconfig wlan0 down
sudo iw dev wlan0 interface add mesh0 type mp mesh_id f6test
sudo ifconfig mesh0 10.100.1.X netmask 255.255.255.0 up
 X represents the number of the VM, e.g., 1 for f6-vm1
sudo iw dev mesh0 set channel 1

Having carried out the commands of the above block on all (virtual) machines you should have a working emulated radio mesh network.
